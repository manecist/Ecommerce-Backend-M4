SUBIDA DE ARCHIVOS DE LA CONSOLA JAVA
SE CREA PACKAGE DE ECOMMERCE CON PRODUCTOS, CARRITO Y TIENDA

* PRODUCTOS TIENE CONSIGO OTROS PACKAGE DE INTERFACES, IMPLEMENTACION Y ENUNCIADOS

-> ENUNCIADOS TIENE CONSIGO ENUM CATEGORIA Y SUBCATEGORIA

-> INTERFACES TIENE INTERFACE IPRODUCTO, IPRODUCTOREPOSITORY, IREGLADESCUENTO

-> PACKAGE DE IMPLEMENTACION TIENE CONSIGO DESCUENTOS Y REPOSITORIO, ADEMAS DE UNA CLASE DE PRODUCTO

--> DESCUENTOS TIENE CLASES DE CATEGORIA, SUBCATEGORIA, POR MONTO Y MANAGER

--> REPOSITORIO TIENE UNA UNICA CLASE LLAMADA PRODUCTOREPOSITORY

** LOS ENUM ESTAN UTILIZADOS DENTRO DE LA CLASE PRODUCTO QUIEN IMPLEMENTA LA INTERFAZ IPRODUCTO
SE CREARON LOS ENUM PARA HACER LA PAGINA MAS MANTENIBLE AL MOMENTO DE DESEAR CREAR OTRA CATEGORIA O SUBCATEGORIA EXTRA SIMPLEMENTE SE AGREGA DENTRO DE LOS ENUM
POR PARTE DEL PROGRAMADOR SIN ESTAR REFACTORIZANDO EL CODIGO, SI NO MAS BIEN AGREGANDO NUEVAS FUNCIONES QUE NO ALTERAN FUNCIONALIDAD

** LAS INTERFACES SON CONTROATOS DE RELACION ENTRE INTERFACE Y CLASS, DONDE PRODUCTO IMPLEMENTS IPRODUCTO, PRODUCTOREPOSITORY IMPLEMENTS IPRODUCTOREPOSITORY, DESCUENTOCATEGORIA,SUBCATEGORIA Y PORMONTO IMPLEMENTS IREGLASDESCUENTO PARA QUE FUNCIONEN DE MEJOR FORMA LOS METODOS Y PUEDAN SER MAS MANTENIBLES EN CASO DE FALLOS O CORRECCIONES.

* CARRITO CONTIENE CLASES CARRITOITEM Y CARRITOSERVISE, ADEMAS CUENTA CON UNA EXCEPTION LLAMADA CANTIDADINVALIDADEXCEPTION, LA CUAL FUNCIONA CUANDO AL AGREGAR UNA CANTIDAD = O MENOR A 0 EN EL CARRITO D EUN PRODUCTO ARRROJA UN ERROR.
  
* TIENDA CONTIENE UNA UNICA CLASE MAIN PARA USO DE CONSOLA TERMINAL JAVA LLAMADA TIENDAAPP DONDE SE IMPLEMENTA EL MENU ADMIN/USUARIO CON SUS RESPECTIVOS PERFILES
